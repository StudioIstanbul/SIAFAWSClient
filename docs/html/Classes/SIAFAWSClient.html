- (void)uploadFileFromURL:(NSURL *)url toKey:(NSString *)key onBucket:(NSString *)bucketName withSSECKey:(NSData *)ssecKey

Parameters

url

local file URL

key

remote AWS key to upload to. If key already exists it will be overwritten without further notice or a new version will be created if versioning is enabled in the bucket settings. Always provide an absolut key path.

bucketName

name of the bucket to upload to

ssecKey

key used to encrypt data, has to be 256 bit and AES256 compatible or nil if upload shall not be encrypted

uploadFileFromURL:toKey:onBucket:withSSECKey:withStorageClass:andMetadata:

upload a file to AWS S3 using a custom key for server side encryption and with metadata. Metadata will be stored in AWS S3 key metadata headers.

- (void)uploadFileFromURL:(NSURL *)url toKey:(NSString *)key onBucket:(NSString *)bucketName withSSECKey:(NSData *)ssecKey withStorageClass:(SIAFAWSStorageClass)storageClass andMetadata:(NSDictionary *)metadata

Parameters

url

local file URL

key

remote AWS key to upload to. If key already exists it will be overwritten without further notice or a new version will be created if versioning is enabled in the bucket settings. Always provide an absolut key path.

bucketName

name of the bucket to upload to

ssecKey

key used to encrypt data, has to be 256 bit and AES256 compatible or nil if upload shall not be encrypted

storageClass

AWS Storage class to be used. May be SIAFAWSStandard or SIAFAWSReducedRedundancy. Direct put to Glacier is not supported by AWS. Set up a lifecycle rule if you want to use Glacier. @see SIAFAWSStorageClass

metadata

NSDictionary containing metadata as strings. NSDictionary keys must not include special characters or white spaces.
